# Fruit Craft (v1.10.10755) - API Request Format

## 1. Authentication & Session
Fruit Craft uses a stateful, cookie-based session architecture rather than standard bearer tokens.

### Initial Auth `/player/load`
The game initially authenticates the device using `/player/load`. The request body is `application/x-www-form-urlencoded` containing:
*   `udid`: Hardware device ID (`system.getInfo("deviceID")`)
*   `device_name`: Hardware name
*   `game_version`: The APK/App version
*   `os_type`: `1` for iOS, `2` for Android
*   `os_version`: OS version string
*   `model`: Device model string
*   `restore_key`: (Optional) Used to link an existing account across devices.
*   `invitation_ticket`: (Optional) Invite code string.
*   `store_type`: e.g. "myket", "cafebazaar", "googleplay"
*   `appsflyer_uid`: Used for marketing analytics.

**Session Cookies:**
Upon a successful `/player/load`, the server responds with HTTP Cookies. The client extracts these using `network.parseResposeHeader` and stores them globally. Every subsequent API request **must** include these Cookies in the HTTP headers to maintain the session.

## 2. Standard HTTP Headers
All API calls (except file downloads) include the following generated by `getDefaultHeader()`:
*   `Content-Type`: `application/x-www-form-urlencoded`
*   `Cookie`: Session identifiers stored from the load request.

## 3. Request Security / Validation Hash
Many critical endpoints (such as `/battle/quest`, `/battle/battle`, or `/player/turnthewheel`) include a validation hash to prevent trivial payload spoofing.

The payload field is defined by `Constants.Player_QKey` which maps to the string key `"check"`.
The client generates an MD5 hash of the player's total quest count (`mainPlayer.q`) as a string, and appends it to the body:
```lua
body["check"] = crypto.digest(crypto.md5, tostring(mainPlayer.q), false)
```
*Note: Because this is predictable, it acts more like a synchronization check rather than strong crypto-auth.*

## 4. Request Payloads
Endpoints expect flat URL-encoded parameters. For complex structures like arrays, the client uses `json.encode()`.
Example `/battle/battle` payload (`newOperation` in `Models.Operation.lua`):
*   `type`: `"battle"`
*   `opponent_id`: Target user ID
*   `attacks_in_today`: Number of times the player attacked this opponent today
*   `cards`: JSON-encoded string of an array containing the IDs of the 4 attacking cards (e.g. `"[1205, 304, 219, 442]"`).
*   `hero_id`: ID of the equipped hero (if any).
*   `check`: MD5 validation string.
